Cisco交换机常用命令

发表于2024-05-16|更新于2024-05-22|路由交换
|阅读量:

>用户模式 # 特权模式
(config)# 全局模式
(config-if)# 接口配置模式
SVI(交换机虚拟接口)：为了远程管理访问交换机。需要给他配备默认网关。个人理解为VLAN接口。
给SVI VLAN99 分配地址172.17.99.11/24
configure terminal
interface vlan 99 //进入SVI(VLAN)的接口配置模式
ip address 172.17.99.11 255.255.255.0 给vlan99分配ip地址
no shutdown //给VLAN配IP得到SVI必须启动才能用
ip default-gateway 172.17.99.1 //配置交换机的默认网关
验证ip设置：
show ip interface brief (全局模式下既可以使用)
//可以用来查看物理接口和虚拟接口的状态

配置交换机端口

configure terminal
interface f0/1 //进入接口配置模式
duplex full //配置接口双工模式
speed 100 //配置接口速率
end //返回特权模式
copy running-config startup-config
//将运行配置保存在启动配置中 这句命令是在特权模式下就可以执行

auto-MDIX 用来自动判断是直通线还是交叉线连接
使用auto-MDIX时必须将接口速率和双工模式都设置为auto,这样才能正常工作
启用auto-MDIX命令需要在交换机的接口配置模式下输入
S1(config-if)#mdix auto;
验证auto-MDIX设置:
show controllers ethernet-controller fa0/1 phy|include MDIX
//|为通道符号，与过滤器include （包含与过滤表达式相匹配的整段信息），section（以过滤表达式开始输出后面的控制信息），exclude（排除与过滤表达式相匹配的所有输出行），begin（从匹配筛选表达式的哪一行开始，显示所有输出行）

#交换机验证命令
//特权模式下才能用show命令。
show interfaces [inferface-id] //显示接口状态和配置
show startup-config //显示当前启动配置
show running-config //显示当前运行配置
show flash //显示有关闪存文件系统信息
show version //显示系统硬件和软件状态
show history //显示输入的命令历史记录
show ip interface //显示接口ip信息
show mac address-table //显示mac地址表

show interfaces常用来检测介质问题，输出的第一个参数(fastethernet0/18 is up)是指的硬件层，表示这个接口是否在接收载波检测信号，第二个参数(line protocol is up)指数据链路层。
interface:up line protocol:down 封装类型不匹配，另一端接口处在错误禁用(error-disabled)状态
interface:down line protocol:down 没连上，或接口另一端可能为管理性关闭状态(administratively down) //该状态可以通过no shutdown手动打开
show interfaces:可以检测输入错误，残帧，小巨人帧，CRC，输出粗我，冲突（半双工），延迟冲突

配置SSH

检查IOS是否支持SSH：show version ，版本名带k9就支持
配置过程：
1.验证是否支持SSH：show ip ssh
2.配置ip域名：ip domain-name name
3.生成RSA密匙对: crypto key generate rsa （删除为：crypto key zeroize rsa）
4.配置用户身份验证：本地身份验证的方法：username username secret password
5.配置VTY路线:
先用line vty range 进入配置vty的模式中
transport input ssh 在vty中启用SSH协议
login local 采用本地认证
6.启动SSH版本2
S1(config)ip ssh version 2

路由器的基本配置

配置路由器接口

1.配置至少一个ip地址 ip address ip-address subnet-mask
2.激活: no shutdown
3.描述:description ……可以为接口配置最多240字符的简短描述

ipv4环回接口

是一个虚拟的逻辑端口，只要路由器运行正常，该接口就会自动处于up状态
Router(config)# interface loopback number //启用环回接口
Router(config-if)# ip address 10.0.0.1 255.255.255.0 //将其配置ip
一个交换机可以不只有一个环回接口

验证直连网络

接口验证命令

show ip(ipv6) interface brief 显示所有接口的汇总信息，包括ip地址和接口的当前运行状态
show running-config interface interface-id:显示应用在指定接口上的命令
show ip route:显示存储在RAM中的ip路由表中的内容，活动接口应当出现在路由表中，而且具有两个相关条目，”C”(已连接)或”L”(本地)来标识

验证接口状态

show ip interface brief 可以查看所有接口的状态
show ip interface fasteither-name 查看某一个接口的状态

验证接口的配置

show running-config interface 显示应用到指定端口上的当前命令
show interfaces 显示所有接口的接口信息和数据包流量的计数
show ip interface 显示路由器所有接口与ip有关的信息

过滤show命令输出

terminal length 数量 可以生成单页更多行，0值可以防止路由器在输出屏幕之间暂停
terminal history size 200 可以设置show history 保存的数量
|为通道符号，与过滤器include （包含与过滤表达式相匹配的整段信息），section（以过滤表达式开始输出后面的控制信息），exclude（排除与过滤表达式相匹配的所有输出行），begin（从匹配筛选表达式的哪一行开始，显示所有输出行）

交换的概念

帧转发

入向：用来描述帧进入设备的端口
出向：用来描述帧离开设备的端口

交换转发方法：1.存储转发交换 2.直通交换
存储转发交换：这种方式会在交换机收到整个帧，并开始使用循环冗余校验CRC的错误检查机制检查这个帧是否存在错误后，再对帧做出转发决策，存储转发交换是思科主要的LAN交换方式。
直通交换：这种方式会在确定了入向帧的目的MAC地址和出向端口之后开始转发。
存储转发交换有两个特征：1.错误检查 和 2.自动缓冲
免分片交换是对直通交换的修正形式，交换机只有读取了数据帧的类型(Type)字段以后才会开始转发这个帧。

缓解网络拥塞

1.高端口速率
2.快速内部交换
3.较大的帧缓冲区
4.高端口密度

VLAN

VLAN的优势

1.广播域更小
2.安全性提升
3.IT效率提升
4.成本降低
5.性能更好
6.项目和应用程序管理更简单

VLAN的分类

默认VLAN，数据VLAN，本征VLAN，管理VLAN，语音VLAN
思科交换机的默认VLAN是VLAN1，对于VLAN1：默认情况下所有端口都被划分到VLAN1，本征VLAN为VLAN1，管理VLAN为VLAN1，VLAN1不能重命名或删除。
本征VLAN是出现在thunk口当中，在access口中是不会有本征VLAN的概念的，本征VLAN就是对于一个未使用VLAN ID标记的帧划分到本征VLAN当中。
管理VLAN是专门为网络管理流量如SSH(端口：22)，Telnet(端口：23)，HTTPS，HTTP,SNMP而配置的数据VLAN。
语音VLAN：
VoIP流量有以下需求：
1.足够的带宽来保证语音质量
2.高于其他网络流量类型的传输优先级
3.可以绕过网络中拥塞区域进行路由
4.跨网络的延迟小于150ms
要满足这些需求：整个网络都必须支持VoIP

多交换机环境下的VLAN

定义VLAN中继

VLAN标记控制信息字段由类型(Type)字段，优先级(Priority)字段，标识符(CFI)，和VLAN ID(VID)字段组成。
类型：2字节，以太网该字段被设置为0x8100
用户优先级：3位
CFI:无用,1位
VLAN ID:12位，最多支持4096个VLAN ID

本征VLAN和802.1Q标记

本征VLAN中对于有标记的帧会直接丢弃，就算这个标记帧的VLAN ID和本征VLAN相同，同样会被丢弃。
对于无标记的帧会转发到本征VLAN当中。

语音VLAN标记：

思科IP电话提供三个端口
端口1.连接到交换机或者其他VoIP设备
端口2.是内部10/100接口，用于承载IP电话流量。
端口3(接入端口).连接到PC或其他设备

连接的IP电话以三种方式中的一种发送语音流量：
1.以标记了第二层服务类型优先级值的语音VLAN发送。
2.以标记了第2层CoS优先级值的访问VLAN发送
3.以无达标的访问VLAN发送。

语音VLAN验证：

show interface(s) fa0/18 switchport (特权模式下查看即可)

VLAN配置：

正常范围VLAN 1-1005，其中1002-1005是为传统网络技术保留的，自动创建，不能删除
VLAN 1也自动创建不能删除。
存储在闪存文件系统中
拓展范围VLAN 1006-4094 存储在运行配置文件中(内存),支持的功能比普通范围的少，且需要配置VTP(VLAN中继协议)透明模式。

VLAN创建命令

configure terminal
vlan vlan-id //使用有效的ID号创建VLAN,并且会进入vlan接口配置模式
name vlan-name //指定vlan的名称
end //返回特权模式

VLAN端口分配命令

configure terminal
interface interface-id //进入接口配置模式
switchport mode access //将接口设置为接入模式
switchport access vlan vlan-id //将端口分配给VLAN
end //返回特权模式

数据VLAN和语音VLAN示例

switchport voice vlan vlan-id：把一个语音VLAN分配给一个端口。
vlan 20
name student
vlan 150
name VOICE
eixt //返回全局配置模式

interface fa0/18
switchport mode access
switchport access vlan 20
mls qos trust cos //用来设置接口的可信状态
switchport voice vlan 150
end
这一个实例告诉我们一个接口可以有两个VLAN
如果交换机不存在VLAN，switchport access vlan 会强制创建一个VLAN

验证VLAN信息

show vlan会显示所有已经配置的VLAN
show vlan brief 逐行显示VLAN名称，状态及其端口
show vlan id vlan-id 显示有关标识的VLAN ID号的信息，对于vlan id，范围是1-4094
show vlan name vlan-name 显示有关表示的VLAN名称的信息
show vlan summary 显示VLAN的汇总信息

show interfaces interface-id switchport和show interfaces vlan vlan-id
前者用于确定端口是否正确划分给了数据vlan和语音vlan

更改vlan端口的成员身份

如果被错误划分到了其他vlan，只需要用
switchport access 正确的vlan 就可以了
要将端口改为默认的VLAN
no switchport access vlan

删除vlan

全局配置模式命令下 no vlan vlan-id 用于从交换机vlan.dat文件中删除一个vlan。
若删除的vlan里面有端口，那么这个端口将会变为inactive，可以手动用no shutdown打开。
在特权模式下可以用delete flash:vlan.dat删除整个vlan.dat文件

VLAN中继

VLAN中继配置命令

configure terminal
interface interface-id
switchport mode trunk //将端口设置为永久中继模式
switchport mode access //将端口设置为永久接入模式
switchport trunk native vlan vlan-id //将vlan vlan-id设置为本征VLAN
switchport trunk allowed vlan vlan-list //指定中继链路上允许的最大VLAN列表

验证中继配置

在端口配置模式中：
show interfaces fa0/1 switchport
//显示交换机S1上端口F0/1的配置
show interface trunk

将中继重置为默认状态

在端口配置模式中：
no switchport trunk allowed vlan删除允许的VLAN，之后会变成允许所有的VLAN
no switchport trunk native vlan 重置中继的本征VLAN，重新使用VLAN 1作为本征VLAN
利用switchport mode access可以将中继特性改为接入特性

动态中继协议(DTP)

DTP为思科专有的协议，可以使得他们自动和相邻设备协商中继，其他厂商的交换机不支持DTP
DTP的默认配置是dynamic auto(动态自动)
switchport nonegotiate 可以关闭DTP
switchport mode dynamic auto 启动DTP

VLAN间路由操作

使用路由将网络流量从一个VLAN转发至另一个VLAN的过程统称为VLAN间路由

VLAN间路由的三个选项：
1.传统VLAN间路由
2.单臂路由器
3.使用SVI进行第三层交换

第三层交换机上的路由

如果想要VLAN可以被其他第三层的设备访问到，这些VLAN必须使用静态或动态路由通告，要在第3层交换机上启动路由，就必须创建一个路由端口。
要在第三层交换机上创建路由端口，需要禁用连接第三层设备的那一个第二层端口的交换机端口特性。
需要在第二层端口上配置no switchport接口配置命令，把他转换为第三层接口，然后就可以在这个接口上使用IPv4进行配置，将其连接到其他第三层设备。

常见的VLAN间路由问题

1.缺失VLAN
如果VLAN不存在则创建VLAN
确保将主机端口划分到正确的VLAN
show vlan[brief]
show interfaces 端口 switchport
ping
2.交换机中继端口存在问题
确保中继配置正确
确保端口是中继端口并且已经启用
show interfaces trunk
show running-config
3.交换机接入端口存在问题
给接入端口分配正确的VLAN
确保端口是一个接入端口并且已经启用
主机配置了错误的子网
show interfaces 端口 switchport
show running-config interface 端口
ipconfig
4.路由器配置存在问题
路由器子接口的IPV4地址配置错误
为路由器子接口分配正确的VLAN ID
show ip interface brief
show interfaces
